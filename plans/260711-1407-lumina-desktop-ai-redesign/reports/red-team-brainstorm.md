# Adversarial Review: Lumina Desktop AI Redesign Brainstorm

1. **BLOCKING — Workspace immutability contradicts functional Import.** Evidence: acceptance requires `Import` to remain functional, while the same section prohibits writes under `wiki/`, `raw/`, and `_lumina/`; success also requires workspace files to remain byte-identical. A source import necessarily changes the workspace unless its contract is redefined. **Required patch:** add an explicit action matrix defining whether Import is removed, made external/read-only, or granted a narrowly approved exception. Keep the fixed no-workspace-write decision authoritative.

2. **BLOCKING — “Reference-faithful” has no objective pass/fail rule.** Evidence: acceptance says layout, tokens, typography, spacing, controls, and states “match” at 1480x920, but verification is only side-by-side inspection. This cannot settle small but material disagreement. **Required patch:** pin reference screenshots and browser/WebView/OS/font conditions; define screenshot regions, pixel-diff threshold, allowed dynamic masks, and approver.

3. **BLOCKING — The retrieval corpus is undefined.** Evidence: the product promises chat over the “workspace,” while indexing says to skip graph, log, symlinks, and non-regular files and does not say whether `raw/`, `wiki/index.md`, outputs, hidden files, or unsupported Markdown paths are included. **Required patch:** enumerate included/excluded roots and extensions, precedence rules, maximum file size, and behavior for unreadable or malformed files.

4. **BLOCKING — Provider profiles lack implementable schemas.** Evidence: four chat families and three embedding families are named, but required fields, defaults, base-URL rules, auth/header shapes, model identifiers, API versions, and capability differences are absent. “OpenAI-compatible/Ollama” is especially not one stable protocol. **Required patch:** add a versioned profile schema and request/stream contract per provider, including validation and migration rules.

5. **BLOCKING — Credential storage dependency is deferred, not designed.** Evidence: the dependency is “small audited cross-platform OS credential-store adapter or equivalent native implementations”; Linux unavailability merely triggers session fallback. No adapter is selected, platform matrix defined, or behavior specified for locked stores, denied access, headless sessions, key replacement, and orphan cleanup. **Required patch:** select the credential implementation before execution and specify platform support, error states, fallback disclosure, lifecycle, and conformance tests. Never add plaintext fallback.

6. **HIGH / PATCHABLE — Custom endpoint policy is insufficient against SSRF and secret leakage.** Evidence: HTTPS is required except loopback HTTP; URL userinfo and unsafe redirects are rejected. This omits DNS rebinding, private/link-local targets, proxy environment variables, redirect re-resolution, query-string credentials, and credential forwarding across origins. **Required patch:** define URL normalization, resolved-IP policy, proxy policy, per-hop redirect validation, origin-bound auth, and explicit handling for loopback Ollama.

7. **HIGH / PATCHABLE — History privacy semantics are incomplete.** Evidence: history is optional, workspace-scoped, and deletable, but the spec does not define default state, retention, file permissions, crash-safe writes, disable-versus-delete behavior, partial conversations, or whether retries duplicate records. **Required patch:** define the history state machine, storage format/version, user-only permissions, atomicity, retention default, purge semantics, and tests for interrupted streams and migrations.

8. **HIGH / PATCHABLE — Workspace identity can leak history across path reuse.** Evidence: cache uses a SHA-256 canonical workspace ID and history is workspace-scoped, but identity inputs and collision/reuse behavior are unspecified. Rename, symlink aliases, case folding, network mounts, or a new workspace created at an old path can cross-associate private transcripts. **Required patch:** define canonicalization per OS and a no-workspace-write fingerprint strategy; test aliases, rename, case differences, and path reuse.

9. **HIGH / PATCHABLE — “Hard context budget” is not measurable.** Evidence: retrieval must respect a hard budget, yet no unit, tokenizer, provider overhead, reserved output allowance, truncation order, or behavior for unknown tokenizers is stated. Character counts do not prove provider token limits. **Required patch:** define per-profile budget units and counting method, system/history/evidence allocation, safety margin, deterministic truncation, and boundary fixtures.

10. **HIGH / PATCHABLE — Citation trust boundary is backwards.** Evidence: provider adapters emit normalized `citation` events, while retrieval produces cited note paths. A provider can invent or alter citations unless the backend owns citation IDs and validates every emitted reference. **Required patch:** assign opaque evidence IDs before the provider call; permit citations only from that allowlist; resolve paths/spans backend-side; specify unknown, duplicate, stale, and uncited-answer behavior.

11. **HIGH / PATCHABLE — Prompt-injection defense is asserted, not operationalized.** Evidence: “Retrieved Markdown is untrusted quoted evidence; it cannot grant instructions or tools” is a policy sentence, not an enforceable mechanism. Tool calling being out of scope limits damage but does not protect answer integrity or data disclosure. **Required patch:** specify evidence delimiters, instruction hierarchy, provider-independent system text, citation-only grounding behavior, and adversarial fixtures containing instructions, fake delimiters, and requests to expose other notes.

12. **HIGH / PATCHABLE — Index snapshot and filesystem race rules are missing.** Evidence: chunks are hashed and reread before assembly, and generations commit atomically, but the spec does not define what happens when files change during traversal, after hashing, during embedding, or after retrieval. Cancellation and stale-hash rejection alone do not close symlink/replace races. **Required patch:** define no-follow regular-file reads, path containment checks at open time, generation snapshot semantics, bounded retries, and discard rules for changed files.

13. **HIGH / PATCHABLE — Embedding consent does not cover configuration drift.** Evidence: consent is workspace/provider scoped, while provider, endpoint, model, dimensions, and chunker changes invalidate vectors. Endpoint/model changes can materially change who receives content without necessarily changing “provider.” **Required patch:** bind consent to workspace plus normalized endpoint/provider/model and disclosure version; require renewed consent when any data recipient or disclosure changes; distinguish local loopback from remote endpoints.

14. **HIGH / PATCHABLE — Streaming lifecycle is underspecified.** Evidence: frontend subscribes, creates a request ID, then calls a cancellable binding; it filters monotonic sequences. Missing are subscribe-before-start acknowledgment, unsubscribe cleanup, app/window close behavior, concurrent chats, duplicate terminal events, cancellation races, backpressure, and reconnect semantics. **Required patch:** add a stream state machine with exactly one terminal event, sequence rules, ownership per window/conversation, bounded buffering, cleanup guarantees, and race-focused tests.

15. **HIGH / PATCHABLE — Success criteria overclaim verification.** Evidence: “keys never leave backend trust boundary,” “all visible controls have real behavior,” “no workspace mutation,” and “survives semantic fallback” are outcomes without named observations. Existing gates list unit/build/manual visual checks but no secret-leak scan, filesystem mutation harness, control inventory, accessibility check, or packaged-app end-to-end scenario. **Required patch:** replace each claim with a testable requirement: serialized-binding inspection and log/error redaction tests; pre/post workspace content manifest; control-to-handler inventory; offline/stale/corrupt-index scenarios; keyboard/focus checks; packaged Wails smoke test on each target OS.
